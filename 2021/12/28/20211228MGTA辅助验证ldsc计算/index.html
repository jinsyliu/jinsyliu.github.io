2.1. 查询使用方法#

usage: mtag.py

mtag: Multitrait Analysis of GWAS This program is the implementation of MTAG method described by Turley et. al. Requires the input of a comma-separated list of GWAS summary statistics with identical columns. It is recommended to pass the column names manually to the program using the options below. The implementation of MTAG makes use of the LD Score Regression (ldsc) for cleaning the data and estimating residual variance-covariance matrix, so the input must also be compatible ./munge_sumstats.py command in the ldsc distribution included with mtag. The default estimation method for the genetic covariance matrix Omega is GMM (as described in the paper). Note below: any list of passed to the options below must be comma-separated without whitespace.（该程序是Turley等人描述的MTAG方法的实现。需要输入以逗号分隔的具有相同列的 GWAS 汇总统计信息列表。 建议使用以下选项手动将列名称传递给程序。 MTAG 的实现利用 LD 分数回归 (ldsc) 来清理数据和估计残差方差-协方差矩阵，因此输入还必须与 mtag 中包含的 ldsc 发行版中的 ./munge_sumstats.py 命令兼容。 遗传协方差矩阵 Omega 的默认估计方法是 GMM（如论文中所述）。 下面注意：传递给以下选项的任何列表都必须以逗号分隔，没有空格。）

2.2. 输入#

Input Files:

Input files to be used by MTAG. The —sumstats option is required, while using the other two options take priority of their corresponding estimation routines, if used.（MTAG 要使用的输入文件。 —sumstats 选项是必需的，而使用其他两个选项时，如果使用，则优先使用它们相应的估计例程。）

Specify the list of summary statistics files to perform multitrait analysis. Multiple files paths must be separated by “,”. Please read the documentation to find the up-to-date set of acceptable file formats. A general guideline is that any files you pass into MTAG should also be parsable by ldsc and you should take the additional step of specifying the names of the main columns below to avoid reading errors.（指定汇总统计文件列表以执行多特征分析。 多个文件路径必须用“,”分隔。 请阅读文档以查找最新的可接受文件格式集。 一般准则是，您传递到 MTAG 的任何文件也应该可以被 ldsc 解析，并且您应该采取额外的步骤来指定下面的主要列的名称以避免读取错误。）

If specified, will read in the genetic covariance matrix saved in the file path below and skip the estimation routine. The rows and columns of the matrix must correspond to the order of the GWAS input files specified. FIles can either be in whitespace-delimited.txt or .npy format. Use with caution as the genetic covariance matrix specified will be weakly nonoptimal.（如果指定，将读取保存在下面文件路径中的遗传协方差矩阵并跳过估计程序。 矩阵的行和列必须对应于指定的 GWAS 输入文件的顺序。 文件可以是 whitespace-delimited.txt 或 .npy 格式。 谨慎使用，因为指定的遗传协方差矩阵将是弱非最优的。）

If specified, will read in the residual covariance matrix saved in the file path below and skip the estimation routine. The rows and columns of the matrix must correspond to the order of the GWAS input files specified. FIles can either be in .txt or .npy format. Use with caution as the genetic covariance matrix specified will be weakly nonoptimal. File must either be in whitespace-delimited .txt or .npy（如果指定，将读入保存在下面文件路径中的残差协方差矩阵并跳过估计例程。 矩阵的行和列必须对应于指定的 GWAS 输入文件的顺序。 文件可以是 .txt 或 .npy 格式。 谨慎使用，因为指定的遗传协方差矩阵将是弱非最优的。 文件必须是空格分隔的 .txt 或 .npy）

2.3. 输出#

Output formatting: Set the output directory and common name of prefix files.（输出格式：设置前缀文件的输出目录和通用名称。）

2.4. 自定义列名称#

Column names of input files:

These options manually pass the names of the relevant summary statistics columns used by MTAG. It is recommended to pass these names because only narrow searches for these columns are performed in the default cases. Moreover, it is necessary that these input files be readable by ldsc’s munge_sumstats command.（输入文件的列名：这些选项手动传递 MTAG 使用的相关汇总统计列的名称。 建议传递这些名称，因为在默认情况下只对这些列执行窄搜索。 此外，这些输入文件必须能被 ldsc 的 munge_sumstats 命令读取。）

2.5. 过滤选项#

Filter Options:

The input summary statistics files can be filtered using the options below. Note that there is some default filtering according to sample size and allele frequency, following the recommendations we make in the corresponding paper. All of these column-based options allow a list of values to be passed of the same length as the number of traits.（可以使用以下选项过滤输入摘要统计文件。 请注意，根据我们在相应论文中提出的建议，根据样本大小和等位基因频率进行了一些默认过滤。 所有这些基于列的选项都允许传递与特征数量相同长度的值列表。）

2.6. 特殊情况#

Special Cases:

These options deal with notable special cases of MTAG that yield improvements in runtime. However, they should be used with caution as they will yield non-optimal results if the assumptions implicit in each option are violated.（这些选项处理显着的 MTAG 特殊情况，这些情况会在运行时产生改进。 但是，应谨慎使用它们，因为如果违反每个选项中隐含的假设，它们将产生非最佳结果。）

2.7. FDR计算#

Max FDR calculation:

These options are used for the calculation of an upper bound on the false disovery under the model described in Supplementary Note 1.1.4 of Turley et al. (2017). Note that there is one of three ways to define the space of grid points over which the upper bound is searched.（这些选项用于在 Turley 等人的补充说明 1.1.4 中描述的模型下计算错误发现的上限。 (2017)。 请注意，可以使用三种方法之一来定义搜索上限的网格点的空间。）
